使用 el-input 时，设置 type=number 时清除右侧样式及限制输入长度
一、清除样式

在使用 el-input 时，想让文本框只能输入数字，可以设置 type="number"，现在就只能输入纯数字文本了，但是，这样的文本框会有一个自带的上下箭头样式，比较难看。可以利用 css 将其去除。我所遇到的问题，将代码写在目标所再页面内的样式标签中并不能生效，于是乎，我将代码写在了 App.vue 的样式标签内（global css）。

  • 可以尝试
    1. 如果写在 目标所在的 .vue 文件中没有生效，可以尝试添加 /deep/ 看看
    2. 建议单独写一个全局的样式表文件，在 main.js 中引入
二、限制文本长度

1.接着说一下，当我们使用 el-input 时，默认是 type="text" 的，这时候我们可以设置官方提供的 maxLength 或 minLemgth 来限制输入的文本长度，但是当 type="number" 时，会发现 max 与 min 失效了，这是因为当我们 type=number 时你时输入不了文本的，这时，可以用一个自定义的事件 oninput 在事件里面获取输入的数字长度，来进行判断，如果大于规定的长度就利用 slice() 进行剪切操作，这样一来便可达到我们需要的效果。

2.如果你仔细观察，你会发现在 type="number" 的文本框内，可以输入字母 e，这是为什么呢? 因为e在数学上代表的是无理数，是一个无限不循环的小数，所以在输入e的时候，输入框会把e当成一个数字看待，那怎么解决这个问题呢，直接看代码

这样一来便可解决可以输入 e 这个bug

三、总结
  1. 添加全局样式，解决 <el-input type="number"/> 右侧箭头样式问题
  2. oninput="if(value.length>6)value=value.slice(0,6)" 自定义事件，解决 <el-input type="number"/> 的输入长度限制
  3. onKeypress="return(/[\d.]/.test(String.fromCharCode(event.keyCode)))" 正则判断，解决 <el-input type="number"/>是可以输入e的bug
