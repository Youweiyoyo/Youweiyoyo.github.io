函数

函数实际上是对象，每个函数都是 Funciton 类型的实例，而 Function 也有属性和方法，跟其他引用类型一样。因为函数是对象，所以函数名就是指向函数对象的指针，而且不一定与函数本身紧密绑定。

  1. 定义函数的几种方式

    1. 命名函数（函数声明）

    2. 匿名函数（函数表达式）

    3. 箭头函数

    4. Fouction 构造函数

      不推荐使用这种语法定义函数，因为这段代码会被解释两次：第一次是将它当作常规 ECMAscript 代码，第二次是解释传递给构造函数的字符串。这样会影响性能

  2. 函数名

    函数名就是指向函数的指针，所以它们跟其他包含对象指针的变量具有相同的行为。这意味着一个函数可以有多个函数名称

    ​ 所有的函数对象都会暴露出一个只读的 name 属性，其中包含关于函数的信息，多少情况下，这个属性中保存的就是一个函数标识符，或者说是一个字符串化的变量名，及时函数没有名称，也会如实显示成空字符串。如果是 Function 构造函数创建的，则会标识成 anonymous “匿名的”

  3. 箭头函数

    很大程度上，箭头函数实例化的函数对象与正式的函数表达式创建的函数对象行为是相同的。任何可以使用函数表达式的地方，都可以使用箭头函数

    1. 箭头函数：如果只有一个参数，那么可以不用写小括号

    2. 箭头函数：没有参数或多个参数时，则需要小括号

    3. 箭头函数：当函数体只有一句的时候也可以不用写大括号

      箭头函数也可以不用大括号，但是这样会改变函数的行为，使用大括号就说明包含 “函数体” ，可以在一个函数中包含多条语句，跟常规的函数一样。如果不使用大括号，那么箭头函数后面就只能有一行代码，比如一个赋值操作，或者一个表达式。而且，省略大括号会隐式返回这行代码的值

    箭头函数虽然语法简洁，但也有很多场合不适用。箭头函数不能使用 arguments 、super 和 new.target ,也不能用作构造函数，此外箭头函数也没有 prototype 属性，且箭头函数自身没有 this 指向，它的 this 是执行上下文中的 this

  4. 函数参数

    Javascript 函数既不关心传入的参数个数，也不关心这些参数的数据类型。在定义函数时要接收两个参数，并不意味着调用时就传两个参数，你可以传一个，三个，甚至一个也不传，解释器也不会报错

    ​ Javascript 函数的参数在内部表现为一个数组。函数被调用时总会接收一个数组，但函数并不关心数组中包含什么。如果数组为空，没有问题；如果数组的元素超出了要求，也没有问题。在使用 function 来定义函数参数时，在函数内部访问 arguments 对象，从中取得传进来的每个参数值

    ​ arguments 对象是一个类数组(伪数组,它不是 Array 的实例),因此可以使用中括号语法访问其中的元素，
